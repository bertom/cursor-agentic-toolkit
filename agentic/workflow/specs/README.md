# Specifications

Technical design documents that translate a feature brief into a concrete implementation plan. Must be approved before implementation begins.

**Naming:** `SP-NNN-<slug>.md` (e.g., `SP-001-user-auth.md`)

**Template:** [`../templates/spec-template.md`](../templates/spec-template.md)

**Created when:** A feature brief has been approved.

**References:** Must link to its parent feature brief (`FB-NNN`).

**Next step:** Create a task pack in `../tasks/` referencing this spec.
